# Plan — 음식점 원산지 표시판·배달앱 메뉴 원산지 체크 글

## 한 줄 scope
배달앱을 같이 쓰는 작은 음식점 사장님이 `매장 메뉴판`, `배달앱 메뉴 설명`, `포장/전단`에서 원산지 표시를 빠뜨리기 쉬운 지점을 점검하게 하는 무료 Tistory 글.

## 독자
- 배달앱을 처음 세팅하는 음식점·분식집·카페 운영자
- 메뉴판은 고쳤지만 배달앱 설명란은 놓친 사장님
- 단속/민원 전에 공식 안내를 다시 확인하려는 소상공인

## 채널/카테고리
- Tistory `업종별 안내문 > 매장 운영·플레이스`
- 무료 글. 판매 CTA 없음.

## 제목 후보
1. 배달앱 쓰는 음식점, 원산지 표시에서 놓치기 쉬운 5가지
2. 메뉴판은 고쳤는데 배달앱은요? 음식점 원산지 표시 체크
3. 작은 식당 원산지 표시, 매장·포장·배달앱 순서로 보기

## 차별화
단순 품목표 나열이 아니라 `실수 사례 → 바로잡는 순서` 구조. 매장 메뉴판만 보는 기존 글보다 배달앱/포장 판매까지 체크하게 한다.

## 이미지 계획
1. Thumbnail: `원산지 표시 / 배달앱도 체크` / 음식점 메뉴판+휴대폰 아이콘 / 오렌지·민트.
2. Explainer: 매장 메뉴판 → 포장 안내 → 배달앱 메뉴 설명 → 공식 안내 재확인 흐름.
3. Action card: 원산지 표시 전 체크 5개 — 품목, 메뉴명, 표시 위치, 배달앱, 변경일.

## 실행 체크리스트
- 농식품부/국립농산물품질관리원 공식 문구와 자료 다운로드 재확인.
- 대상 품목·과태료 금액·표시방법은 최신 공식 안내 기준으로만 표현.
- 10개 comparable rhythm 반영. 최근 세금 글의 `한눈에 보면→체크리스트→공식 안내` 반복을 피하고, 실수 사례부터 시작.
- 이미지 3장 생성 및 local visual QA.
- Tistory 카테고리·HTML 렌더링·공개 visual QA는 backlog job으로 넘김.
