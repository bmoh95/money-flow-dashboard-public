메뉴판 원산지는 고쳤는데, 배달앱 메뉴 설명은 그대로 둔 경우가 있습니다.

작은 식당에서 은근히 생기는 실수입니다. 매장에 붙인 표시판만 생각하다가 포장 안내, 전단, 배달앱 화면을 따로 못 보는 거죠.

이 글은 원산지 표시 품목을 전부 외우자는 글이 아닙니다. 사장님이 공식 안내를 확인하기 전에, 어디부터 봐야 하는지 순서를 잡는 글입니다.

먼저 이런 실수가 많습니다

  • 새 메뉴를 추가했는데 원산지 표시판에는 반영하지 않음
  • 매장 메뉴판에는 적었지만 배달앱 메뉴 설명에는 빠짐
  • 고기·김치·쌀처럼 자주 쓰는 재료 변경일을 메모하지 않음
  • 거래처 안내만 보고 공식 표시 방법은 다시 확인하지 않음
  • 포장 주문 안내문과 매장 게시판 내용이 서로 다름

여기서 중요한 건 “매장 안에 붙였는지” 하나가 아닙니다.

손님이 주문 전에 실제로 보는 화면에 원산지 정보가 보이는지까지 같이 봐야 합니다.

1. 매장 메뉴판만 보면 빠질 수 있습니다

원산지 표시는 손님이 주문 전에 확인할 수 있는 방식으로 안내되어야 합니다.

그래서 매장 안 메뉴판이나 게시판만 보고 끝내면, 배달앱이나 포장 안내에서 빠지는 부분이 생길 수 있습니다.

요즘은 손님이 매장에 들어오기 전에 배달앱 화면에서 먼저 메뉴를 봅니다. 그 화면에 원산지 정보가 보이는지도 같이 봐야 합니다.

2. 메뉴가 바뀐 날을 따로 적어둡니다

원산지는 거래처나 재료가 바뀌면 같이 확인해야 합니다.

예를 들어 김치, 고기, 쌀처럼 자주 쓰는 재료는 늘 같은 곳에서만 들어온다고 생각하기 쉽습니다.

그런데 임시 품절이나 대체 납품이 생기면 표시 내용도 다시 봐야 합니다.

복잡한 장부가 아니어도 됩니다.

메뉴명 / 재료 / 현재 표시 / 변경일 / 확인한 사람 정도만 메모해도 나중에 훨씬 편합니다.

3. 배달앱 메뉴 설명란을 따로 봅니다

배달앱은 매장 메뉴판과 별도로 수정해야 하는 경우가 많습니다.

신메뉴를 올릴 때 가격과 사진은 바로 바꾸는데, 원산지 문구는 예전 상태로 남는 일이 있습니다.

메뉴별 설명란, 가게 공지, 원산지 표시 전용 입력칸이 있는지 차례로 확인해보세요.

앱마다 화면 이름이 다르니, 특정 버튼명보다 손님 화면에서 보이는지를 기준으로 보는 게 낫습니다.

4. 공식 안내로 마지막 확인을 합니다

블로그 글이나 거래처 안내문만 보고 끝내면 위험합니다.

음식점 원산지 표시 대상, 표시 방법, 예외는 공식 안내에서 다시 확인해야 합니다.

농림축산식품부, 국립농산물품질관리원, 지자체 안내를 기준으로 보세요. 특히 과태료 금액이나 대상 품목은 글이 낡을 수 있어서 숫자를 단정하지 않았습니다.

5. 출력용 표시판보다 “변경 루틴”이 더 중요합니다

원산지 표시판을 예쁘게 만드는 것보다 더 중요한 건 바뀔 때 놓치지 않는 루틴입니다.

가게에서 이렇게 정해두면 좋습니다.

  • 새 메뉴 등록 전 원산지 확인
  • 거래처 변경 시 원산지 표시판 확인
  • 배달앱 메뉴 설명 동시 수정
  • 포장 안내문/전단 내용 확인
  • 월 1회 손님 화면에서 직접 점검

한 번에 완벽하게 만들려고 하면 오히려 미룹니다.

손님이 보는 화면 순서대로 하나씩 확인하는 게 더 현실적입니다.

출처와 확인할 곳

  • 농림축산식품부 · 음식점/배달음식 원산지 표시 관련 안내
  • 국립농산물품질관리원 · 원산지 표시 종합 안내
  • 각 지자체 식품위생/원산지 표시 안내 페이지

실제 표시 전에는 최신 공식 자료를 꼭 다시 확인하세요.

정리

원산지 표시는 매장에 붙였는지만 보면 부족할 수 있습니다.

배달앱을 쓰는 가게라면 손님이 보는 화면을 기준으로 매장, 포장, 앱 메뉴 설명을 같이 봐야 합니다.

제일 안전한 방법은 메뉴가 바뀌는 날 원산지 메모도 같이 바꾸는 것입니다.
